Phase - Overview

Phase stands for Pentaho Analysis Editor. With Phase you can create and edit data models that are required by Pentaho Analysis.

Pentaho Analysis: some background

Pentaho Analysis is a back-end component of the Pentaho Platform that provides Online Analytical Processing capabilities (OLAP). OLAP provides multi-dimensional analysis of business data.

Multi-dimensional analysis

Multi-dimensional analysis categorizes business data in two kinds of items:

Measures
Quantative data that are indicative of the performance of the business process. For example, typical measures for a sales process are units sold, profit, cost.
Dimensions
Qualitative data that describes situational aspects of the business process and provide the context for the measures: the who, what, when, where and why. These different types of descriptive data are grouped into a number of independent dimensions. For example, typical dimensions for a Sales process are the customer and the employ that handled the sale (the who), the product (what), order date or shipping date (when), the store (where) and maybe a promotion campaign that was in effect (why).

In multi-dimensional analysis, the measure data is grouped over its corresponding dimensions and aggregated. Dimensions themselves are hierarchically structured into a number of distinct levels. For example, an order date dimension may have year, month and day levels, and a store dimension may have a geographically oriented hieararchy with country, state, city and address levels.

This way of presenting the data allows users to navigate through large datasets and to project, filter and compare measures along with whatever aspects of the business process described by data from the dimensions.

The Mondrian engine

The heart of Pentaho Analysis is the Mondrian engine. This component is typically not directly visible to end users. But user-facing applications like Pentaho Analyzer and Saiku Analytics rely on mondrian to support the data analysis process. Other pillars of the Pentaho platform, like Pentaho reporting and dashboarding can also use Pentaho Analysis as a data source.

From the point of view of the applications and components that rely on Mondrian, it does 2 different things:

  • Present a multidimensional view (i.e., a set of measures and corresponding dimensions) of business data. This allows applications to compose analytical queries.
  • Accept and process analytical queries, and return multi-dimensional datasets.

Analytical queries are formulated in the multi-dimensional expression language (MDX), which allows business data to be expressed as a selection of measures and whatever items from any corresponding dimensions. Mondrian then does the hard work of performing the correct aggregations on the measure data, and structures the result in a way that reflects the requested dimensional data to describe the context of the measure aggregates.

Typically these queries and their results are not visible directly to an end-user. Rather, end-user tools provide a graphical user interface that presents the dimensional model of the business data. The application allows its end-users to select and arrange items from the dimensional model in some way that makes sense with respect to the business problem they want to solve. The application then generates a corresponding MDX query, and visualizes the query result in some way the user understands.

Schemas

Mondrian is a so-called Relational OLAP-engine (ROLAP). This means the actual data that makes up the measures and dimension data are sourced from a live relational database.

In order to perform its tasks, Mondrian needs to know how to connect to the relational database backend. It also needs to know how the tables and columns provided by the relational backend map to items in the relational model that mondrian presents. This mapping is defined in a so-called schema.

One mondrian schema is associated with one database connection but can define many different measures and dimensions. Mondrian schemas are typically expressed in specially designed XML-dialect.

Where Phase fits in

Phase provides a graphical user interface that allows users to edit and create Mondrian schemas. Phase is a thin client application that runs inside the Pentaho User console.

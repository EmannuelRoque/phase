Database Treeview

The database treeview is located on the left bottom of the phase application below the Model Treeview. Phase will automatically update the database treeview to reflect the database connection that is associated with the schema to which the currently selected node in the model treeview belongs. This happens either when a node is selected in the model treeview, and also when the value of the Datasource field in the schema editor is changed.

Database Treeview Node Types

The following list explains the different node types that may apper in the database treeview:

Catalog node
Catalog nodes represent a database catalog.
Schema node
Schema nodes represent a database schema. Database schema nodes can either be top-level nodes, or may appear beneath a database catalog node.
Tabe node
Table nodes represent a database tables.
View node
View nodes represent database views.
Column node with an integer datatype.
Column with a tinyint, smallint, integer, or bigint datatype.
Column node with an decimal datatype.
Column with a numeric or decimal datatype.
Column node with an floating point datatype.
Column with a real, double or float datatype.
Column node with an string datatype.
Column with a varchar, nvarchar, nclob, nchar, longvarchar, longnvarchar, clob, or char datatype.
Column node with an binary string datatype.
Column with a varbinary, longvarbinary, blob, or binary datatype.
Column node with an date datatype.
Column with a date datatype.
Column node with an time datatype.
Column with a time datatype.
Column node with a datetime datatype.
Column with a datetime datatype.
Column node with a bit or bool datatype.
Column with a bit or bool datatype.
Column node with Java Object datatype
Column with a struct, rowid, ref, null, other, distinct, datalink or Array datatype.
Column node with XML datatype
Column with a struct, rowid, ref, null, other, distinct, datalink or Array datatype.
Column node with other datatypes

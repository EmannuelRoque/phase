Node types

Nodes in the model treeview are of a particular type and represent either a mondrian schema or an element within such a schema. The nodes have both a label and an icon. The icon indicates the type of modelelement that is represented by the node. The name indicates the identifiers (the name) of the model element that the node represents.

The following list explains the different node types that may apper in the model treeview:

Schema node
A schema node represents a Pentaho Analysis Model - (a Mondrian Schema).
Cube node
A cube node represents a Cube. A cube is a domain-specific collection of measures and corresponding dimensions. Cube nodes can be found beneath schema nodes.
Shared Dimension node
A shared dimension node represents a shared dimension. A shared dimension is essentially a so-called conformed dimension which can be used by multiple cubes. Shared dimension nodes can be found beneath schema nodes.
Private Dimension node
A private dimension node represents a private dimension. A private dimension is a dimension that is specific to just one particular cube. Shared dimension nodes can be found beneath cube nodes.
Dimension Usage node
A dimension usage node represents a dimension usage. A dimension usage is essentially a role playing dimension: it refers to a shared dimension but is given a specific role and boud to a specific context within a cube. Dimension usage nodes can be found beneath cube nodes.
Hierarchy node
A hierarchy node represents a hierarchy within a shared or private dimension.
Level node
A level node represents a level within a hierarchy.
Measure node
A measure node represents a measure. Measure nodes can be found beneath cube nodes. Measure nodes are leaf nodes.
Calculated Member Node
A calculated member node represents a calculated member. Calculated member nodes can be found beneath cube nodes and virtual cube nodes. Calculated member nodes are leaf nodes.
Named Set Node
A named set node represents a named set. Named set nodes can be found beneath cube nodes, virtual cube nodes, and also schema nodes. Named set nodes are leaf nodes.
Virtual Cube Node
A virtual cube node represents a virtual cube. Virtual cube nodes can be found beneath schema nodes.
Cube Usage node
A cube usage node represents a Cube that is referenced somewhere in a Virtual Cube. Cube usage nodes can be found beneath virtual cube nodes.
